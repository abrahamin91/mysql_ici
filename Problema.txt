Como el jefe de sistemas de información para el Colegio Whylie estas a cargo del desarrollo de un nuevo sistema de registro de estudiantes. Al Colegio le gustaría un nuevo sistema cliente-servidor para reemplazar su sistema más viejo desarrollado en torno a la tecnología mainframe. El nuevo sistema permitirá a los estudiantes registrarse a los cursos y ver la boleta de calificaciones de los ordenadores personales conectado a LAN del campus. Los profesores podrán acceder al  sistema para inscribirse para impartir cursos, así como registrar las calificaciones.

Debido a la disminución del financiamiento federal, el colegio no puede permitir reemplazar todo el sistema de una vez. El colegio mantendrá la existencia de la  base de datos del catálogo de cursos, donde se mantiene toda la información de los cursos. Esta base de datos es una base de datos de relación Ingres corriendo sobre DEC VAX. Afortunadamente el colegio ha invertido en la interface SQL abierta que permite acceder  a esta base de datos desde los servidores Unix del colegio. El rendimiento del sistema legado es bastante pobre, así el nuevo sistema debe garantizar que el acceso a la información del sistema legado ocurra de manera oportuna. El nuevo sistema deberá acceder a la información de los cursos desde la base de datos legada pero no actualizarla. La oficina de registros continuara manteniendo la información de los cursos a través de otro sistema. 
Al inicio de cada semestre, los estudiantes pueden solicitar un catálogo de cursos que contiene una lista de oferta de cursos para el semestre. Información de cada curso, tales como profesores, departamento, y prerrequisitos, se incluirán para ayudar a los estudiantes a tomar decisiones informadas.

El nuevo sistema permitirá a los estudiantes seleccionar cuatros cursos ofertados para el próximo semestre. Adicionalmente, cada estudiante indicara dos opciones alternativas en caso de que el estudiante no puede ser asignado a la primera elección. Los cursos ofertados tendrán que ser máximo de diez estudiantes y mínimo de tres. Un curso ofertado con menos de tres estudiantes será cancelado. Para cada semestre,  hay un periodo de tiempo donde los estudiantes pueden cambiar su horario. Los estudiantes serán capaces de acceder al sistema durante este tiempo para añadir o eliminar cursos. Una vez que el proceso de registro esta completado para un estudiante, el sistema de registro envía información al sistema de facturación para que el estudiante se pueda facturar para el semestre. Si un curso está lleno durante el proceso actual de registro, los estudiantes serán notificados del cambio antes de la presentación del horario para el procesamiento.
Al final del semestre, el estudiante será capaz de acceder al sistema a ver la boleta de calificaciones electrónica. Ya que las calificaciones del estudiante es información  sensible, el sistema deber emplear medidas de seguridad extras para prevenir  el acceso no autorizado.

Los profesores deberán poder indicar al sistema en línea que cursos impartirán. También necesitaran ver que estudiantes se inscribieron a sus cursos ofertados. Adicionalmente, los profesores podrán registrar las calificaciones para los estudiantes de cada clase.
